# uptime命令
> 查看linux系统的负载信息。

##命令格式
uptime [参数]
##命令参数
* -V: 显示版本信息

##命令结果
命令 `uptime`

```
[suoper@prod-java-bigdata-26 ~]$ uptime
 17:39:34 up 46 days, 16:37,  2 users,  load average: 3.51, 3.03, 2.90
``` 
###结果说明
* 17:39:34 当前时间
* up 46 days, 16:37 目前运行多长时间，46天16小时37分
* 2 users 用户连接数，是总连接数而不是用户数
* load average: 3.51, 3.03, 2.90 系统平均负载，统计最近1分钟、5分钟、15分钟的系统平均负载

##平均负载
> 是指在特定时间间隔内运行队列中的平均进程数。
> 如果每个CPU内核的当前活动进程数不大于3的话，那系统运行是良好的。如果每个CPU内核的任务数大于5的话，那么系统的性能有严重的问题
